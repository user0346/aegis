Was AEGIS macht

Layered Defense. Jede Stufe blockt was die vorherige durchgelassen hätte.

🛡

Pre-Execution Blocking

Hash-Blacklist, Filename-Patterns, Byte-Pattern-Detection. Eine bekannte Malware wird quarantänisiert bevor Windows sie öffnet.

🧠

Selbst-Lernend

Pattern-Calibration aus deinen Approve/Deny-Entscheidungen. Claude-gestützte Reflektion. Encrypted Persistent Memory.

📡

Network-Watcher

Live-Connection-Logging, Portscan- und Flood-Detection, DNS-Sinkhole gegen Tracker und IP-Logger.

⚙

Autonomy-Levels

Off / Observe / Suggest / Auto / Full — Pin-geschützt mit TTL-Session. Auto-Demote bei kritischen Threats.

🔒

Self-Protection

SHA-Pin aller eigenen Files. Defender-Exclusion-Watchdog. Hosts-File-Wiederherstellung. Niemand modifiziert AEGIS unbemerkt.

🪶

Background-Service

Läuft beim Boot ohne UI. UI kann geschlossen werden, Schutz bleibt aktiv. Auto-Restart bei Crash.

Schritt 1

Anthropic Console öffnen, Account anlegen oder einloggen.

console.anthropic.com ↗

Schritt 2

Unter "API Keys" einen neuen Key erstellen. Beginnt mit sk-ant-.

Schritt 3

In AEGIS-UI → Settings → "Anthropic API-Key" einfügen → Speichern. Wird DPAPI-verschlüsselt lokal abgelegt.

Ist das sicher?

Dein Key wird nie über die UI-Bridge weitergegeben, nie geloggt, nie ans Anthropic-Backend geschickt außer im offiziellen Messages-API-Call. Die Speicherung läuft über Windows DPAPI (User-Account-gebunden) — selbst andere User auf der gleichen Maschine können den Key nicht entschlüsseln.

AEGIS Browser-Extension

Der Browser ist die häufigste Quelle für Schädlinge. Die Extension schützt dich on-page — auch wenn AEGIS-Service nicht läuft.

🚨 IP-Logger blockieren

Bekannte Grabify-, IPLogger-, 2no.co-Domains werden vor dem Klick erkannt. Warnung im Tab + Block-Option.

🔗 URL-Scanner

Jeder Link wird gegen Phishing-/Punycode-/Subdomain-Spoofing-Patterns geprüft. Visueller Indicator.

📥 Download-Hinweis

Wenn eine .exe/.scr/.bat von einer fragwürdigen Domain downloaded wird, springt die Extension ein bevor der Browser sie ablegt.

🔄 Sync mit AEGIS

Wenn AEGIS-Service läuft, syncen Block-Listen + Decisions. Was die Extension blockt, lernt AEGIS — und umgekehrt.

Installation

  1. Browser-Extension-Ordner aus dem AEGIS-ZIP holen (AEGIS/extension/)
  2. Chrome/Edge öffnen: chrome://extensions bzw. edge://extensions
  3. "Entwicklermodus" einschalten
  4. "Entpackte Erweiterung laden" → den extension/-Ordner auswählen
  5. Fertig — AEGIS-Icon erscheint in der Toolbar

Security-Prinzipien

Encrypted at Rest

Alle Secrets (API-Keys, Owner-Pin, gelernte Patterns) liegen DPAPI-verschlüsselt unter ~/.aegis/secrets.bin und ~/.aegis/memory/*.bin.

Consent-Framework

Jede potentiell verändernde Aktion (Web-Search, Shell, Learning-Write) verlangt HMAC-signierte One-Shot-Token. Audit-Log immutable.

Integrity-Verify

SHA-Pin aller AEGIS-eigenen Files. Boot-Time-Compare. Mismatch → Safe-Mode + Autonomy-Disable.

Signed Updates

Manifest mit Ed25519-Signature. Downgrade-Attacks werden erkannt. AEGIS lädt nie selbst — Owner entscheidet.

Pipe-DACL

IPC-Pipe nur für current User + LocalSystem. Token-Auth. JSON-Schema-Validation auf jedem Command.

Hard-Blacklist

Auch FULL-Autonomy kann nie: Defender modifizieren, Audit-Log löschen, AEGIS-Code ändern, Passwörter ändern, Firewall deaktivieren.
